## Overview

Welcome to the Foxtrot-Four Solutions (F4S) Manual—a comprehensive, professional-grade collection of Standard Operating Procedures (SOPs) for all non-combat gameplay loops in Star Citizen 4.5. This manual represents the operational doctrine of the F4S corporation and is designed to provide new and experienced operators with detailed, ship-specific guidance for maximum efficiency and profitability.

## Structure

The F4S Manual is organized by **Division** and **Gameplay Loop**, with each ship receiving its own dedicated SOP chapter.

```
foxtrot-four-solutions/
├── docs/
│   ├── sentinel/          # Mining, Salvage, Trade
│   ├── nexus/             # Medical, Cargo, Repair

│   └── obsidian/          # Illicit (future)
├── data/                  # Reusable databases (JSON)
├── assets/                # Images, diagrams, flowcharts

```

## Naming Convention

All F4S documents follow the **F4S-STD-03** naming standard:

```
F4S_[DIVISION]_[MISSION]_[ASSET]_v[VERSION].html
```

**Example:** `F4S_SEN_MIN_PROSPECTOR_v1.html`

- **PREFIX:** F4S (Corporate Identifier)
- **DIVISION:** SEN (Sentinel), NEX (Nexus), VAN (Vanguard), AEG (Aegis), OBS (Obsidian)
- **MISSION:** MIN (Mining), SAL (Salvage), MED (Medical), LOG (Logistics), REF (Refueling), CBT (Combat), SAR (Search & Rescue)
- **ASSET:** Ship Name (e.g., PROSPECTOR, MOLE)
- **VERSION:** v1, v2, etc. (allows for patch updates without overwriting)

## How to Use

1. **Select Your Division:** Navigate to the appropriate division folder (e.g., `sentinel/` for mining).
2. **Open the SOP:** Click on the HTML file for interactive viewing, or use the Markdown version for GitHub.
3. **Follow the Procedures:** Each SOP is divided into tabs covering Logistics, Loadouts, Strategy, Execution, and Checklists.
4. **Reference the Data:** Use the JSON databases in `/data/` for component specifications, vendor locations, and ore profitability.
